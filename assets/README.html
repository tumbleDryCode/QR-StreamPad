This page includes the QR code generator below.
Basically Your setting a delimited string as the QR Code string.
This string is then read by the app and the data is tokenized for the database or viewing.
In this demo the | is used to split each record and the semi-colon is used to split the fields.
The android apps sqlite database is pretty simple with 1 table that has 7 basic fields:
rec_id, rec_type, rec_val_a, rec_val_b, rec_val_c, date_added and date_modified. see UtilSQLiteAdapter and UtilDBRecord.
The rec_id is androids _id column and the rest are used in the delimted strings below.
The rec_type determines if its a leaflet, recipe, etc.
 rectypes
5 - leaflet ; id ; title; ..
10 - recipe ; id ; title ;..
50 - product ; promo id ; title ; pricing ; favorite..
When using this for testing, dont forget to change the id paramater 5;300 ... 5;301 ..  5;302 .. etc.,
or it will read records with that id already inserted in db.
